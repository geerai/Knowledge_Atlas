Track FOUR: Interaction Design & Evaluation

Understand how different user types navigate Knowledge Atlas. Analyze their information needs, improve interaction design, and measure whether the system actually works for them.

Track Overview

Knowledge Atlas is complex. We need to personalize it for different user types because a student explorer's journey is very different from an instructor's, or a researcher's, or a theory explorer's. Your track is about understanding those differences and making the system better for each user type.

The Goal: Map out how different user types navigate the K-Atlas system. Identify missing CTAs, dead ends, information overload, and misalignments between user needs and page structure. Then test improvements with real users and measure success.

User Types We Support

Student Explorer
Learning the field. Wants overview of topics, propositions, evidence. Navigates by curiosity.
Researcher
Testing hypotheses. Needs deep access to evidence, methodologies, caveats. Searches by topic/methodology.
Contributor
Adding to K-Atlas. Needs to understand schema, propose new warrants, upload evidence. Oriented toward creation.
Instructor
Teaching with K-Atlas. Needs to understand propositions deeply, curate content for courses, track student engagement.
Practitioner
Applying K-Atlas. Wants actionable guidance on built environment design decisions based on evidence.
Theory Explorer
Studying epistemology. Interested in warrant structures, evidence dynamics, coherence metrics.

Core Skills You'll Develop

  • User Research: Conducting interviews and creating personas for different user types
  • Journey Mapping: Visualizing the path a user takes through K-Atlas to accomplish their goal
  • Information Architecture: Understanding what content each user type needs and when
  • Cognitive Load Analysis: Identifying where pages are overwhelming or confusing
  • Usability Testing: Running tests with real users, measuring task completion rates
  • Interaction Design: Improving CTAs, navigation, search, filtering, content structure

Weekly Syllabus

Weeks Phase Activities & Deliverables
1–2 User Type Analysis Interview or create personas for all 6 user types. Document: What questions does each user have? What's their goal? What frustrates them?
3–4 Journey Mapping Map current user journeys for each type. Where do they get stuck? Where are missing CTAs? Review docs/USER_JOURNEY_AUDIT_2026-04-01.md.
5–6 Information Architecture Audit For each user type: What content do they need at each step? Is cognitive load too high? Where are dead ends? Design improvements.
7–8 Prototype & Implement Implement CTA improvements, navigation fixes, content reorganization. Create improved page layouts or features. Commit to git.
9–10 Usability Testing & Iteration Test with real users (5-8 per user type). Measure task completion rates. Document findings. Iterate on design. Course-wide integration.
Starting Point: There's already a user journey audit at docs/USER_JOURNEY_AUDIT_2026-04-01.md in the repo. Start there to see what gaps have already been identified.
Key Metrics:
  • Task Completion Rate: % of users who complete their goal without getting lost
  • Time to Goal: How long does it take to accomplish the core task?
  • Cognitive Load: Do users feel overwhelmed or confused?
  • Information Findability: Can users find the content they need?
  • CTA Effectiveness: Do navigation buttons and links guide users to the right place?
