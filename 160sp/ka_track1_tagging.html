COGS 160 · Track 1 of 4

Track ONE: Image Tagging

Develop annotation tools that label architectural and environmental images with evidence-based tags aligned with K-ATLAS propositions. Learn data verification and pipeline architecture.

Your Tasks This Quarter

Here are examples of the tasks students in this track will work on:

What you'll learn: image classification systems, evidence-based annotation schemas, data quality verification, and pipeline architecture. You'll develop a Python tool that helps other students annotate images consistently and verify that annotations align with K-ATLAS propositions.

Available Tag Bundles to Work On

Choose one or more tag bundles below. Each bundle represents a specific category of architectural/environmental features that students need to label across images.

Lighting Conditions
ENVIRONMENTAL
Natural light, artificial light, daylighting quality, shadows, glare assessment
Spatial Layout
ARCHITECTURAL
Room configuration, furniture arrangement, wayfinding cues, visual hierarchy
Material & Color
AESTHETIC
Surface textures, color palettes, material properties, aging and wear patterns
Accessibility Features
FUNCTIONAL
Ramps, handrails, doorways, circulation paths, clear floor space requirements
Track Milestones:
  • Weeks 1–2: Article collection (assignment A0) + tag bundle selection
  • Weeks 3–4: Build annotation tool prototype + write tag schema documentation
  • Weeks 5–6: Annotate 100 images, verify consistency, refine tool
  • Weeks 7–8: Final tool polish, documentation, handoff to fall cohort
  • Weeks 9–10: Course-wide integration and Fall 160 site debugging
