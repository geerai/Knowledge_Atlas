COGS 160 · Track 2 of 4

Track TWO: AI Article Pipeline

Build automated article discovery systems using AI. Implement RAG-based retrieval, relevance scoring, and Value of Information prioritization to continuously find and filter research articles.

Track Overview

This track is not about manually finding more citations. It's about automating the article discovery process using artificial intelligence and information theory.

The Goal: Build a system that continuously hunts for relevant papers and presents them to other students ranked by how much they'd contribute to filling knowledge gaps. You'll combine Retrieval-Augmented Generation (RAG), relevance filtering, and Value of Information (VOI) scoring.

Core Skills You'll Develop

  • Automated Retrieval: Query engines that connect to Semantic Scholar API and other open-access sources
  • RAG Systems: Retrieval-Augmented Generation pipelines that rank articles by relevance to existing knowledge
  • Relevance Filtering: Distinguish experimental articles from theory papers and meta-reviews
  • Value of Information: Prioritize which articles would most advance the knowledge base
  • Quality Assurance: Automated gates that verify article type, topic relevance, and data quality
  • Benchmarking: Test and measure retrieval accuracy against human-selected articles

Weekly Syllabus

Weeks Phase Activities & Deliverables
1–2 Article Collection (A0) Upload 10 experiment articles + 10 best articles from Google Scholar (everyone does this)
3–4 Basic Query Engine Connect to Semantic Scholar API. Build query builder. Retrieve 50 candidate articles for assigned topics. Write data validation spec.
5–6 RAG & Relevance Scoring Implement RAG using embeddings. Build relevance classifier (experiment vs. theory). Test on 100 articles. Measure F1 score.
7–8 VOI & Prioritization Integrate Value of Information weighting. Rank articles by likely contribution to gaps. Build automated quality gates. Benchmark against manual selection.
9–10 Final Pipeline & Integration Polish pipeline, write operational documentation, integrate into Fall 160 site. Course-wide debugging.
Key Concepts to Understand:
  • RAG: Retrieval-Augmented Generation — combining vector search with LLMs to rank articles contextually
  • Relevance: Not just "mentions keywords," but "is an empirical study on topic" vs. "is a review or theory paper"
  • Value of Information: Which articles would fill the most critical gaps in the knowledge base?
  • Semantic Scholar API: Open-access, powerful tool for academic article retrieval
  • Embedding-based search: Using language models to understand semantic similarity between papers
