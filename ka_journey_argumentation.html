◧What this page displaysthe information on offer

The argumentation layer would render Dung's (1995) abstract argumentation frameworks over the Atlas's claim corpus. Nodes are arguments — a claim plus its supporting reasons. Edges are attacks: one argument defeats another. The computation produces an acceptability labelling that sorts every argument into IN (accepted), OUT (rejected), or UNDEC (undecided), under a chosen semantics (grounded, preferred, or stable).

Where the EN shows epistemic support, the argumentation layer shows argumentative structure — which argument defeats which, and which coherent sets of arguments survive mutual attack. It makes contested claims visible as contested, without pretending to resolve the contest.

The page does not yet exist. What follows is a spec — the right target for a future-quarter T4 redesign loop whose deliverable is an informed specification, not an implementation.

△Why this page is hard to designthe challenges

  1. Semantics is a choice, not a default. Grounded, preferred, and stable semantics give different IN/OUT labellings; the page must let the user pick, and must explain what the choice means, without trapping the user in a philosophy-of-argumentation lecture.
  2. Attack vs. disagreement. Not all disagreement is an attack in Dung's sense. Mapping corpus disagreement onto the attack relation is an extraction decision the page must surface.
  3. Scale. At corpus scale, the attack graph is dense; visualising it is the same density problem as the EN, with an added layer of per-argument labelling that changes with semantics.
  4. Explanation. A user who sees "this argument is OUT" needs to see why — which attackers defeat it, and whether those attackers are themselves defeated. Recursive explanation is a distinctive requirement.
  5. No familiar metaphor. Unlike a belief network (which people can read as a mind-map) or a topic page (which people can read as a category browser), an argumentation framework has no everyday metaphor. The page must invent one.
